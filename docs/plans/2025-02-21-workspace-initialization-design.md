# 工作目录初始化设计文档

## 背景

在 setup 窗口中，用户选择工作目录并点击"确认并开始"后，需要初始化该工作目录。初始化包括创建 `.plan-claude` 标记文件夹，用于标识这是 plan-desktop 的工作区。

## 目标

- 在用户确认工作区时，自动在工作目录创建 `.plan-claude` 文件夹
- 处理各种边界情况（已存在、权限问题等）

## 方案

### 架构

在主进程的 `confirm-workspace` IPC 处理器中增加工作目录初始化步骤：

```
confirm-workspace 处理器
  ├── 验证工作区路径存在
  ├── 创建 .plan-claude 文件夹（如失败则返回错误）
  ├── 保存配置到 store
  ├── 关闭 setup 窗口
  └── 创建主窗口
```

### 行为定义

| 场景 | 行为 |
|------|------|
| 目录不存在 | 创建文件夹，继续 |
| 目录已存在 | 静默忽略，继续 |
| 权限不足/创建失败 | 返回 `{ success: false, error: '...' }`，阻止窗口切换 |
| 路径无效 | 返回错误，保持 setup 窗口 |

### 错误处理

- 创建失败时返回具体错误信息（如 "无法创建标记目录，请检查写入权限"）
- SetupView.vue 显示错误消息，用户可修改目录后重试

## 实现位置

- `electron/main.ts`: `confirm-workspace` IPC 处理器
- `src/views/SetupView.vue`: 错误消息展示
