# 概览页面重新设计

## 背景

当前概览页面（MainView.vue）仅展示一个欢迎卡片和4个静态统计卡片，信息展示不够充分。需要增强页面以显示工作目录的文件结构和文件大小。

## 目标

1. 在概览页面添加文件结构展示功能
2. 显示每个文件的大小，自动选择合适的单位（B/KB/MB/GB）
3. 采用标签页切换的方式组织内容（概览/文件）
4. 文件树支持展开/折叠，按字母排序

## 设计方案

### 布局结构

采用内嵌式标签页布局：

```
┌─────────────────────────────────────────────────────┐
│ 工作区图标  │  顶部标题栏                │ ⏱ 运行中 ▼ │
│ ────────── │ ───────────────────────────────────────── │
│            │  ┌───────────┬───────────┐                │
│ 📊 概览   │  │  📊 概览  │  📁 文件  │ ← 标签页      │
│            │  └───────────┴───────────┘                │
│            │                                          │
│            │  [当前激活标签页的内容区域]               │
│            │                                          │
└────────────┴──────────────────────────────────────────┘
```

### 标签页内容

#### 概览标签页（默认）

- 欢迎卡片：显示工作区名称和路径
- 统计卡片网格（2x2布局）：
  - 待办任务：12（进行中，较上周 +3）
  - 已完成：28（本周，较上周 +8）
  - 文档：45（总计，本周新增 5）
  - 项目：6（活跃，2个即将截止）

#### 文件标签页

- 目录树组件展示工作目录结构
- 每个节点显示：图标 + 文件名 + 文件大小
- 支持展开/折叠文件夹
- 排序规则：文件夹在前，文件在后，均按字母 A-Z 排序
- 文件大小自动选择单位（B/KB/MB/GB）

示例：
```
📁 src/
  📁 components/
    📄 Button.vue (15.2KB)
  📄 main.ts (2.5KB)
  📄 App.vue (8.7KB)
📄 package.json (1.8KB)
📄 README.md (4.2KB)
```

### 技术实现

#### 组件选择

- 使用 naive-ui 的 `NTabs` + `NTabPane` 实现标签页切换
- 使用 naive-ui 的 `NTree` 或自定义组件实现目录树

#### 数据流

1. 文件树数据通过 Electron IPC 从主进程获取
2. 主进程使用 Node.js `fs` 模块递归读取目录
3. 计算每个文件的大小，构建树形数据结构
4. 渲染时按字母排序文件夹和文件

#### IPC 通信

新增 IPC 通道：
- `get-directory-structure`: 请求目录结构
- 参数：`{ path: string }`
- 返回：`TreeNode[]`

### 样式规范

遵循 DESIGN.md 的瑞士极简主义深色主题：

- 标签页使用 naive-ui 默认样式，保持深色主题一致性
- 文件树最大高度：`calc(100vh - 280px)`，超出时内部滚动
- 字体：14px，行高 1.6
- 颜色使用 CSS 变量：`--color-bg-base`、`--color-text-primary` 等

## 边界情况处理

1. **空目录**：显示提示文本"暂无文件"
2. **权限不足**：显示错误提示，跳过无权限访问的目录
3. **大量文件**：使用虚拟滚动优化性能（如文件数 > 500）
4. **文件大小计算**：符号链接不计入大小，显示链接目标大小
